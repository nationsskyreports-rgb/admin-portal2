Authentication
How admin login works
Type username + password
→
db.from('admins').select('*').eq('username').eq('password_hash')
→
Save to sessionStorage as 'nos-admin'
→
Redirect to index.html
ScenarioWhat happens
Open any page without sessioncheckAuth() → redirect to login.html
Open page with valid sessioncheckAuth() returns session object → page loads
Click logoutsessionStorage.removeItem('nos-admin') → redirect to login
Session data{ username, role, loginTime }
Password storagePlain text in admins.password_hash — change via Supabase directly
⚠️ Security note: Admin password is stored as plain text in the admins table. Anyone with the service_role key can read it. Keep the GitHub repo Private at all times.

ActionWhat it does
Create WeekInserts new row in schedule_weeks with status=Draft
Save ScheduleUpserts all rows in schedule (conflict on agent_id+shift_date)
Publish1. Saves schedule 2. Sets week status=Published 3. Auto-generates breaks 4. Updates annual_leave 5. Shows to agents

Publish flow in detail: For each working day in the week, the algorithm calculates Break 1, Lunch, and Break 2 for each agent — ensuring minimum 2 agents always on queue. Times are saved to both break1/2/lunch AND scheduled_break1/2/lunch.
Annual leave update: On publish, counts all Annual days per agent per month for the entire year → updates used_jan...used_dec + total_used + remaining in annual_leave table.

Breaks — breaks.html
/admin-portal2/pages/breaks.html
▼
ActionWhat it does
Daily ViewShows all agents with their break times for selected date. Queue timeline shows coverage.
Save AllUpserts breaks for all agents on selected date. Saves BOTH actual AND scheduled columns.
Copy from YesterdayLoads yesterday's breaks as starting point. Saves to actual columns only — scheduled columns NOT affected (known limitation).
Regenerate from ScheduleRe-runs the break algorithm based on today's shifts. Overwrites everything.
Weekly ViewShows break times across the full week for all agents.
Queue Timeline: Shows how many agents are available at each 15-minute slot. Red = below minimum (2 agents), Yellow = at minimum, Green = OK. Algorithm ensures breaks are spread out.
⚠️ Known limitation: "Copy from Yesterday" does not copy scheduled_break columns. Scheduled columns only get populated via Save All and Regenerate. This means break change comparison won't work for copied days unless you then Save All.

KPIs — kpis.html
/admin-portal2/pages/kpis.html
▼

Three tabs: KPI Table, Leaves, Annual Leave.

TabWhat it showsCan edit?
KPI TableAll agents with Conformance, Missing, AHT, Calls, Exceptions, Quality for selected monthOnly Quality score is editable here
LeavesSpecial leave days (double pay, etc.) from leaves tableAdd/delete leave records
Annual LeaveAnnual leave balance per agentEdit entitlement and carry-over
KPI calculation (same as Agent Portal):
• Conformance = average of daily_performance.conformance
• Missing Time = sum of missing_sec → if ≤300sec = "Full Time"
• AHT = average of avg_aht where avg_aht > 0
• Exceptions = count of approved excuses × 2 hours
• Quality = quality_scores.score for that month
Copy KPI Report: Button at top copies a formatted text summary to clipboard — useful for sharing with management.

xCally Import — xcally-import.html
/admin-portal2/pages/xcally-import.html
▼

The main way to get performance data into the system. Run this monthly (or more often).

Export CSV from xCally
→
Drop CSV here
→
Process → Review table
→
Save All → daily_performance
→
KPIs update automatically
StepDetails
CSV parsingReads each row, matches agent by xcally_name field. If name doesn't match → row is skipped with warning.
CalculationPer agent per day: active_sec, missing_sec, avg_aht, calls count, conformance %
Target hoursSet once (fixed) or per date range. Used to calculate conformance = active/target × 100.
SaveUpserts to daily_performance — conflict on agent_id + perf_date. Safe to re-import.
Reapply ExcusesAfter import, click this to adjust missing_sec for approved excuses (deducts excuse time from missing).
⚠️ Name matching: The agent's xcally_name in the agents table must match EXACTLY the name in the CSV. If an agent's name doesn't match, their data won't import. Fix in Agents page → Edit → xCally Name field.

Adherence — adherence.html
/admin-portal2/pages/adherence.html
▼

Two sections: Deviation tracking (manual) + Break Changes Report (automatic comparison).

Deviation TypeMeaning
Late LoginAgent started shift later than scheduled
Early LogoutAgent ended shift earlier than scheduled
Long BreakBreak lasted longer than allowed
Missed ShiftAgent didn't show up at all
Extra BreakAgent took an unscheduled break
Break Changes Report (new): Compares scheduled_break1/2/lunch vs actual break1/2/lunch. Shows all differences. If diff ≥ 15 minutes → flagged with option to manually add as deviation. This ensures fairness — admin decides, not automatic.
Waiving: Any deviation can be waived (excused). Go to Waiving page to view/undo all waivers.

Maintenance Guide
Everything you need to keep the system running
🔄 Update the code
1. Edit the file locally
2. Push to GitHub (admin-portal2-main repo)
3. Cloudflare auto-deploys in ~2 min
4. Hard refresh browser (Ctrl+Shift+R)
🔑 Change admin password
Go to Supabase Dashboard
→ Table Editor → admins
→ Edit the password_hash column
(plain text, no hashing)
➕ Add new admin user
Supabase → Table Editor → admins
→ Insert row with username, password_hash, role
📊 Monthly KPI routine
1. xCally Import → upload CSV
2. Click "Reapply Excuses"
3. Quality page → add reviews
4. Check KPIs page for accuracy
🔧 Fix wrong KPI data
Re-import the CSV for that date range
Upsert is safe — won't duplicate
Or edit directly in Supabase Table Editor
📅 Weekly schedule routine
1. Create new week in Schedule page
2. Fill all shifts
3. Publish → breaks auto-generated
4. Check Breaks page for accuracy
☕ Daily breaks routine
Breaks page → select today
Review the queue timeline
Adjust if needed → Save All
Check Break Changes Report at end of day
⚠️ Agent name mismatch
If xCally import skips an agent:
1. Check the CSV for exact name spelling
2. Agents page → Edit → xCally Name
3. Re-import the CSV
